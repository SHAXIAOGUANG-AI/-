# 正式版 V1 开发计划

## 当前已经完成

- 原型升级为正式版 V1 骨架。
- 增加订单详情、认证中心、协议流程、平台工作台。
- 抽出独立模拟数据，后续可以替换成云数据库。
- 发布需求增加企业名称、企业联系人、服务费预估和审核前提。
- 接单流程增加订单详情和申请接单入口。
- 完整版 V2 已增加云开发配置、前端服务层和云函数骨架。
- 默认使用本地模拟数据，开通云环境后可切换到云函数。

## 正式上线前必须补齐

1. 用户体系
   - 微信登录
   - 手机号绑定
   - 实名认证
   - 角色：工作者、代班者、平台审核员、企业联系人

2. 订单体系
   - 发布需求
   - 企业授权审核
   - 代班者申请
   - 双方确认
   - 履约签到
   - 完成确认
   - 评价与投诉

3. 合规与风控
   - 企业授权材料
   - 劳务协议模板
   - 电子签
   - 短期保险
   - 私下交易风控
   - 岗位敏感权限限制

4. 支付与结算
   - 微信支付
   - 平台信息服务费
   - 工资托管或企业线下结算
   - 退款和异常处理

5. 后台管理
   - 审核订单
   - 审核用户
   - 查看履约状态
   - 处理异常和投诉
   - 数据统计

## 建议技术路线

- 小程序端：微信原生小程序。
- 后端：微信云开发，适合第一版快速上线。
- 数据库：云开发数据库。
- 文件存储：云存储，用于企业授权材料、协议附件。
- 云函数：处理订单状态、审核、通知、服务费计算。
- 后台：先做小程序内工作台，后续再做 Web 管理后台。

## 数据集合建议

- users：用户资料和认证状态。
- companies：企业与授权联系人。
- orders：代班需求订单。
- applications：接单申请。
- agreements：协议记录。
- insurance_policies：保险记录。
- payments：支付和结算记录。
- reviews：评价和履约记录。
- risk_events：异常和风控事件。

## 重要提醒

这个业务涉及劳动关系、劳务协议、岗位授权、保险和企业管理边界。正式运营前，必须让律师或专业人事合规人员确认业务模式和协议文本。

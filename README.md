# 纸乔代班小程序原型

这是一个微信小程序正式版 V1 骨架，用于演示“企业授权短期代班/短期劳务撮合”的产品思路。

## 需要的软件

1. 微信开发者工具
   - 用来打开、预览和调试小程序。
2. 微信小程序账号
   - 原型阶段可以先用测试号。
   - 真正上线时需要注册主体并完成审核。
3. 后端和数据库
   - 原型暂时不需要。
   - 正式版需要保存用户、订单、协议、保险、支付、评价等数据。

## 如何打开

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择当前文件夹：`zhiqiao-daiban-miniprogram`。
4. AppID 可以先选择测试号或使用游客模式。
5. 导入后点击“编译”预览。

## 当前功能

- 首页：展示项目定位、合规前提、认证、协议和平台工作台入口。
- 发布：工作者填写代班需求、企业联系人、服务费预估，并提交审核。
- 接单：代班者查看订单列表，进入订单详情并申请接单。
- 订单详情：展示岗位要求、费用、保险和合规检查。
- 认证中心：展示手机号、实名认证、技能标签和企业授权材料流程。
- 协议流程：展示授权、协议、保险和履约跟进流程。
- 平台工作台：展示审核、协议、履约和异常处理入口。
- 我的：展示账户状态、风险控制和正式版软件要求。
- 云开发：已预留登录、提交订单、申请接单、审核订单、确认接单、协议记录、风控记录、履约更新和支付占位云函数。

## 演示模式与云开发模式

默认是演示模式，数据来自本地 `data/mock.js`。

开通微信云开发后，修改 `config/env.js`：

```js
cloudEnabled: true,
cloudEnvId: "你的云开发环境ID"
```

然后部署 `cloudfunctions` 目录下的云函数即可接入真实数据库。

## 生产提醒

`createPayment` 目前只是支付占位函数，不会真正扣款。真实上线前需要微信支付商户号、支付回调、退款逻辑和财务对账。

## 重要提醒

这个产品方向存在劳动合规和企业管理风险。建议正式推进时把“顶包”改成“企业授权短期代班”或“合规短期劳务撮合”，并且只允许以下场景：

- 企业或公司明确同意。
- 双方签订劳务协议。
- 为代班者配置短期保险。
- 平台记录沟通、履约和评价。
- 不支持私下替岗或绕过企业授权。
